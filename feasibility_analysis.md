# 抖音评论采集器 - 可行性分析与路线图

## 1. 项目概况
**目标**: 采集过去3天内与“复刻台球杆”相关的抖音评论，筛选包含购买意向关键词（“怎么买”、“价格”）的内容。

## 2. 可行性分析

### 2.1 技术可行性
*   **搜索与发现**: 可行。抖音网页版支持关键词搜索。
    *   *挑战*: 原生不支持“过去3天”筛选。通常只有“最新”、“最多点赞”或“过去1天”、“过去一周”。
    *   *解决方案*: 使用“过去一周”筛选，后期处理时间戳，或如果每日运行则使用“过去1天”。
*   **评论提取**: 可行。
    *   *挑战*: 评论动态加载（无限滚动）。
    *   *挑战*: 反爬虫（验证码、登录要求）。
*   **意向筛选**: 可行。简单的关键词匹配。

### 2.2 反爬虫措施 (风险: 高)
*   **签名**: API请求需要 `X-Bogus` 和 `_signature` 参数。
*   **验证码**: 滑块验证码频繁出现。
*   **IP封禁**: 高频请求会导致IP被封。
*   **登录**: 查看大量评论通常需要登录。

### 2.3 法律与道德 (风险: 中)
*   采集公开数据通常处于灰色地带。
*   “复刻”商品可能敏感。
*   **建议**: 仅用于个人分析，严禁转售数据。

## 3. 技术路线选项

### 路线 A: Web自动化 (推荐 - 稳定性高)
*   **工具**: Python + Playwright / Selenium。
*   **优点**: 易于绕过 `_signature` (浏览器自动处理)。易于处理验证码 (可暂停等待人工干预)。
*   **缺点**: 速度较慢，资源占用高。
*   **工作流**:
    1.  启动浏览器 (无头或有头模式)。
    2.  搜索 "复刻台球杆"。
    3.  按时间筛选 (过去一周)。
    4.  遍历视频结果。
    5.  打开视频，滚动评论。
    6.  提取文本和时间戳。
    7.  按关键词筛选 ("怎么买", "价格")。
    8.  保存有效线索。

### 路线 B: API 逆向工程 (维护成本高)
*   **工具**: Python + Requests + Node.js (用于JS执行) 或 RPC。
*   **优点**: 速度快，轻量级。
*   **缺点**: `X-Bogus` 算法频繁变更，难以维护。

## 4. 推荐路线图 (路线 A)

1.  **环境搭建**: Python, Playwright。
2.  **原型开发**: 编写脚本打开抖音，搜索并打印视频标题。
3.  **数据采集**: 实现视频列表滚动和详情页解析。
4.  **评论解析**: 实现评论加载和关键词筛选。
5.  **优化完善**: 添加错误处理、重试逻辑和数据持久化 (CSV/SQLite)。

## 5. 预估成本
*   **开发时间**: 1-3天完成功能原型。
*   **资源**: 本地机器或小型VPS。代理 (可选但推荐)。
